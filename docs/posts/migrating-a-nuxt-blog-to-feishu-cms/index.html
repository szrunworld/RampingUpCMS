Migrating a Nuxt Blog to a Feishu CMS Workflow

A real Feishu Base record plus Feishu Doc body, wired to match your current Nuxt blog API shape.

这是一篇真实放在飞书文档里的示例文章，用来验证你的博客前端可以把飞书当成 CMS 内容源。

Why This Fits Your Current Site

Your current Nuxt blog already reads list and detail data from an API, so the frontend does not need a full rebuild.

We can keep the page routes and components, and only replace the content source behind them.

Recommended Content Split

  • Use Feishu Base for metadata like title, slug, SEO fields, cover, status, and doc token.
  • Use Feishu Docs for the article body so your team can edit content without touching code.
  • Keep a small adapter layer that transforms Feishu data into the exact JSON shape your current pages expect.

What This Demo Proves

This document is linked from a Base record and then rendered through the local adapter preview.

Once that works, the production site can switch from the old blog API to the Feishu-backed adapter with minimal frontend changes.
